# Landing Narrative

Treat the docs landing as a versioned worldview, not a neutral product brochure.

For each governance version:

- restate what became safe to trust
- state what is still intentionally unresolved
- keep the copy emotionally charged enough that the landing feels inhabited
- avoid generic compliance language unless a contract explicitly requires it

The landing must present `v0.7.9` as kernel-complete and `v0.7.10` as docs-evolution territory.
Tie every narrative move back to the current release truth rather than aspirational marketing.
