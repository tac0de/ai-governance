# Landing Narrative

Treat the docs landing as a versioned worldview, not a neutral product brochure.

For each governance version:

- restate what became safe to trust
- state what is still intentionally unresolved
- make the copy feel inhabited, not committee-written
- avoid generic compliance language unless a contract explicitly requires it

`v0.8.0` closes as a mission-control release: the control plane is sharper, the commercial tiers are explicit, and the landing finally behaves like a live system surface.
Tie every narrative move back to release truth rather than aspirational marketing.
