# 74. MCP Taskforce (Minimal)

## Purpose
Adopt MCP only when it directly improves product KPI, without increasing governance risk.

## Why Taskforce (not full department)
- Current stage is product validation.
- Full MCP department increases operational overhead too early.
- Small taskforce keeps decisions reversible.

## Entry Criteria (all required)
1. Clear KPI link (`activation`, `retention`, `conversion`, or `ops time saved`)
2. Bounded cost estimate (monthly cap defined)
3. Security boundary defined (data scope, token scope, revocation path)

## Operating Rule
- One MCP at a time.
- 2-week trial window.
- Keep if KPI improves and risk remains bounded.
- Remove if no measurable gain.

## Required Deliverables per MCP Trial
- one-page goal
- risk checklist
- rollback checklist
- post-trial result note
