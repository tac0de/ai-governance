# 중앙통제 거시 플랜모드 운영표준 v0.1 (한국어 간편판)

목적: 사람은 목표만 주고, 에이전트는 구체화/검증/보고를 수행하도록 운영을 단순화한다.

## 핵심 원칙
- 중앙통제(`ai-governance`)는 실행 엔진이 아니라 정책/승인/검증/감사 책임을 가진다.
- 서비스는 실제 MCP 호출과 런타임 실행을 담당한다.
- 새 MCP 필요 시 서비스가 요청하고, 중앙 승인 후 allowlist/registry 반영한다.
- 보편 MCP만 중앙 표준으로 생산하고, 서비스 특화 MCP는 서비스 어댑터로 우선 구현한다.

## 사람(오너)이 하는 일
- 목표 한 문장 + 금지사항 + 리스크 티어 + 완료조건 + 롤백조건 제시
- 범위 변경, 리스크 상향, 검증 실패 시 최종 의사결정

## 에이전트가 하는 일
- 목표를 거시 플랜으로 구조화
- 필요한 intent/evidence 문서 자동 생성
- 검증 스크립트 실행 및 실패 복구
- 최종 보고를 1~4 포맷으로 제출

## 기본 사이클 (권장)
1. 목표 입력 (`tmp/pm_objective_*.txt`)
2. 플랜 패키지 생성 (`scripts/macro_plan_pack.sh`)
3. 실행/수정
4. `bash scripts/validate_all.sh` 검증
5. 최종 보고(무엇/어디/사람확인/위험)

## 문서 정책 (간소화)
- 사람용: 한국어 브리프 1개(쉽게 읽히는 문장)
- 기계용: 최소 JSON evidence 1개(경로+sha256)
- 중복 보고서는 archive로 이동하고 핵심 문서만 최신 유지

## 중앙통제 vs 오케스트레이션 차이
- 중앙통제: "무엇이 허용되는가"를 결정
- 오케스트레이션: "어떤 순서로 실행하는가"를 수행
- 운영상 최적점: 중앙은 작고 엄격하게, 서비스 실행은 빠르고 유연하게
