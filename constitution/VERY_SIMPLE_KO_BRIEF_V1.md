# 아주 쉬운 거버넌스 한 페이지 v1

## 1. 이 시스템이 하는 일
- 규칙대로 일하는지 확인하는 시스템입니다.
- 아무나 마음대로 실행하지 못하게 막습니다.
- 실행 결과를 나중에 다시 확인할 수 있게 남깁니다.

## 2. 역할은 딱 이렇게만 씁니다

### 정부 레이어 (안쪽 통제)
- StrategyBoard: 큰 방향 정함
- PolicyOffice: 규칙 만듦
- AuditCore: 규칙 위반/통과 판정

### 기업 레이어 (바깥 실행)
- CEO: 전략 책임 (인간+AI 공동, 최종 결정은 인간 CEO)
- CTO: 실행 책임 (운영 실행 단일 창구)
- HeadOfProduct: 제품 방향
- HeadOfRevenue: 유통/매출 방향

## 3. 절대 규칙
- 정부 역할과 기업 역할을 섞으면 안 됩니다.
- 정책 확인 없이 실행하면 안 됩니다.
- 감사 확인 없이 배포하면 안 됩니다.

## 4. 지금 당장 기억할 것
- 역할을 더 늘리지 않습니다.
- 문서/정책/실행 권한표를 같은 역할셋으로 맞춥니다.
- v1.1부터 실행 역할 이름은 CTO로 고정합니다. (구 명칭 ControlPlaneOperator는 과거 v1.0 기록에서만 허용)
- 최종 승인권은 인간이 가집니다.
