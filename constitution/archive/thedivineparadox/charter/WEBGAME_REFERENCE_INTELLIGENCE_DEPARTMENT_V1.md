# Webgame Reference Intelligence Department v1

## Purpose
Collect, validate, and provide benchmark-grade web/mobile web game references so design and product decisions are evidence-backed.

## Scope
- Own reference acquisition pipelines for UI, loop design, and monetization patterns.
- Own reference quality scoring and tagging standards.
- Own reference artifact publishing for release planning.

## Inputs
- Benchmark questions from Game UI Department and Game Level Design Department.
- Engine capability constraints from Game Engine Department.
- Policy constraints for compliant, non-deceptive experience design.

## Outputs
- Versioned reference packs (`images`, `interaction notes`, `loop snapshots`).
- Benchmark digest for each release milestone.
- Traceable reference IDs for governance and audit use.

## Department Rules
- Must store source URL, capture timestamp, and relevance reason for each reference.
- Must include at least one desktop and one mobile-web reference per benchmark topic.
- Must not publish references without provenance metadata.

## Coordination Boundary
- Works with Game UI Department for visual and interaction benchmarking.
- Works with Game Level Design Department for loop and progression benchmarking.
- Works with Game Engine Department for render and technical feasibility benchmarks.
- Exposes references through approved interfaces (MCP optional, not mandatory by policy).
