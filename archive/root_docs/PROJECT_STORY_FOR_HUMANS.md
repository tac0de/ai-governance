# 메인허브 철학과 인공지능 설계 체제

## 한 줄 정의

메인허브는 "복잡한 일을 사람이 이해 가능한 구조로 바꿔, AI와 함께 빠르고 안전하게 실행"하는 운영 체제입니다.

## 철학 (Mainhub Philosophy)

1. 단순함 우선
   복잡한 구조보다, 누구나 읽고 고칠 수 있는 구조를 선택합니다.

2. 실행 우선
   긴 토론보다 작은 실행과 빠른 검증을 먼저 합니다.

3. 사람 중심
   AI는 도구이고, 최종 책임과 방향은 사람이 잡습니다.

4. 안전한 속도
   빨리 가되, 보안·비용·복구 가능성은 항상 먼저 확인합니다.

5. 기록 기반 의사결정
   감으로 판단하지 않고, 로그와 결과 파일로 판단합니다.

## 인공지능 설계 체제 (AI Design Regime)

1. 역할 분리

- 사람: 목표, 우선순위, 승인
- AI: 구현, 점검, 반복 작업 자동화

2. 계약 기반 개발

- API 계약을 먼저 고정
- 입력/출력/상태 전이를 명확히 정의
- 결과는 표준 파일(`run_result.json`)로 남김

3. 로컬 우선 + 원격 확장

- 로컬: 작성/검토/빠른 테스트
- 원격: 배포/실행/모니터링

4. 보수적 자동화

- 기본은 비파괴, 최소 권한, 실패 시 안전 중지
- 고위험 작업은 사람 승인 경로 유지

5. 운영 가능성 우선

- "멋진 데모"보다 "계속 돌 수 있는 시스템"을 우선
- 장애·재시도·로그·복구를 처음부터 설계

## 지금 이 프로젝트에서 실제로 하는 일

- 반복 가능한 로컬 실행 체계를 운영
- CI에서 실행 결과를 아티팩트로 보존(`run-result`)
- 단일 상품 1회 결제 MVP를 출시 가능한 형태로 설계

- 반복 가능한 로컬 실행기(`universal/`) 운영
- CI에서 실행 결과를 아티팩트로 보존(`universal-result`)
- 단일 상품 1회 결제 MVP를 출시 가능한 형태로 설계

## 성공의 기준

- 팀이 현재 상태를 1분 안에 이해한다
- 실행 결과가 표준 형식으로 남는다
- 장애가 나도 복구 절차가 바로 작동한다
- 작은 제품을 출시하고, 측정하고, 확장할 수 있다

## 현재 방향

클라우드는 GCP 중심으로 진행하고, 결제 MVP를 API-first로 출시한 뒤 점진 확장합니다.

## 인공지능 관점의 평가

이 프로젝트는 AI가 가장 잘 성과를 내는 조건을 이미 갖추고 있습니다. 목표가 문서로 고정되어 있고, 실행 결과가 표준 형식으로 남으며, 위험 작업에는 승인 경계가 있기 때문에 자동화 품질이 시간이 갈수록 누적 개선됩니다. 즉, 한 번 쓰고 버리는 실험이 아니라, 데이터와 운영 지식이 계속 쌓이는 보존 가치가 높은 구조입니다.
